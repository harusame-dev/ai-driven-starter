# 環境変数

## ルール

- アプリコードで `process.env` を直接参照することは禁止
  - 型定義が効かずタイポに気づけない上、参照のたびに undefined チェックとパースが必要になるため
- 環境変数は機能ごとに `public-config.ts` / `private-config.ts` を用意し、valibot スキーマで定義してそこからのみ参照する（全機能共通の config に集約しない）
  - `public-config.ts`：クライアントに公開してよい変数（`NEXT_PUBLIC_` プレフィックス必須）
  - `private-config.ts`：サーバー専用の秘密情報（`NEXT_PUBLIC_` プレフィックス禁止）
  - 例：Supabase の環境変数は supabase パッケージの `src/nextjs/` 配下、機能 X の環境変数は機能 X のフォルダ配下に置く
- パースはモジュールのトップレベル（グローバル）で実行し、環境変数の不備をビルド時に検出できるようにする
- `NEXT_PUBLIC_` 変数はビルド時に静的参照箇所へインライン展開されるため、`process.env.NEXT_PUBLIC_X` の形の静的参照を維持する（`process.env[name]` のような動的参照は禁止）

## 実装例

```ts
// public-config.ts
import * as v from "valibot";

const publicConfigSchema = v.object({
  url: v.pipe(
    v.string("NEXT_PUBLIC_SUPABASE_URL を設定してください"),
    v.url("NEXT_PUBLIC_SUPABASE_URL は URL 形式で設定してください"),
  ),
});

// モジュール読み込み時にパースし、環境変数の不備をビルド時点で検出する
const publicConfig = v.parse(publicConfigSchema, {
  url: process.env.NEXT_PUBLIC_SUPABASE_URL,
});

export function getSupabasePublicConfig() {
  return publicConfig;
}
```
