# テスト戦略（Next.js）

[テスト戦略.md](./テスト戦略.md) の方針を Next.js プロジェクトに適用するうえでの、Next.js 固有の取り扱いをまとめる。

## サーバーコンポーネントの扱い

### 方針

- Next.js のサーバーコンポーネント（React Server Components）は **E2E テストで動作を担保** する
- サーバーコンポーネントのファイルは **カバレッジ計測の対象に含めない**
- サーバーコンポーネント内のロジック（データ取得・データ整形・条件分岐・計算など）は **極力外部ファイルの関数に切り出し、切り出した関数をテスト対象とする**

### 実装方法

#### ロジックの切り出し

サーバーコンポーネントのファイルには「データ取得（`fetch` や DB アクセス）」「切り出した関数の呼び出し」「JSX の出しわけ・レンダリング」のみを残し、ロジックは別ファイルに分離する。

#### E2E での担保

- サーバーコンポーネントが描画されるページに対しては、主要なユーザーシナリオの中で表示・遷移を E2E で確認する
- サーバーコンポーネント固有の表示分岐（ログイン状態による出し分け、権限による出し分けなど）は E2E のシナリオに含める
