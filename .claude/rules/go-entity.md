# エンティティ

## ファクトリー関数

- `New<Entity>` は新規作成用のファクトリー関数とし、 ID 採番・初期値設定・不変条件のチェックなど新規作成に必要なロジックを含める
- 既存のエンティティ（DB から取得したデータなど）の復元には `Restore<Entity>` を別途用意し、引数の値をそのままフィールドに設定するだけのロジックを含まない関数とする
- 呼び出し意図（新規作成 / 復元）をシグネチャと関数名で明示する
- パッケージ名がエンティティ名と一致する場合（例: `authors` パッケージの `Author`）は、呼び出し側の stutter（`authors.RestoreAuthor`）を避けるため `<Entity>` を省略して `New` / `Restore` とする
  - 汎用名パッケージ（`commands` / `queries` など）や複数のエンティティを含むパッケージでは、`New` だけでは対象が曖昧になるため省略しない（`commands.NewCourse` は維持）
