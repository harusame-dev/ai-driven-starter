# UI・UX

## 絵文字の禁止

アイコンライブラリ（lucide）を使用する

## コンポーネント実装
0 から実装するのは避け shadcn/ui（base-ui） のコンポーネントを優先する

## スクロールバーガターの CLS 防止 

スクロール箇所に stable-gutter-stable を設定する

## クラス名の結合

テンプレートリテラルはさけ、必ず cn 関数を使う

## tailwind 優先

style props や css クラスの使用は控え、className で tailwind クラスの使用を優先する

## デザインシステム

任意のカラーの指定は控え、デザインシステムのカラーを優先する
特別な意味や意図のあるカラーはコメントで意図や理由を補足すること

## クラス名の抽象化を避ける

複数箇所で使われるデザインの共通化は css のクラスや中間変数を使うのではなく、コンポーネントとして切り出す

## クラス名の直接指定

特別な理由がなければ中間変数や定数でクラスを持つことは避け、className に直接記載する

例外例：表示要素を配列で持って要素ごとに指定したいクラスを持つ場合
